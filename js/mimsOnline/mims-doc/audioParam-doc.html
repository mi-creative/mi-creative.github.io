Audio-rate Parameter Module



@{name} param @{input_name}

This creates a named floating point parameter that can be used throughout the model script to control any desired value.
It is an audio-rate parameter, controlled by the inlet number of @{input_name} (e.g. @in2). There is no initial value, as the parameter is assigned to the audio stream soon as the simulation starts running.

Examples:

# Create a named parameter for stiffness and damping
@audioK audioParam @in1
@audioZ audioParam @in2

# Create a harmonic oscillator that uses these parameters
# (no @ used when using the parameters as arguments)

@o1 osc 1 audioK audioZ 0. 0.8

# the stiffness and damping parameters are now controlled by the
# first and second audio inlets of the object.

Position Input Module

@{name} posInput {position}
This creates a position input from the outside world (considered as a material point). It must be named in[1,2,...] as this will define the inlet number of the generated gen~ or Faust dsp object.
Position may be expressed either as a scalar (e.g. 1) or a vector in brackets (e.g. [1 0 2]), in which case the z coordinate serves as the base position for the 1D physics and x-y serve to position the material point in a rendered 3D space.
Examples:
@in1 posInput 0.
@in2 posInput [0 4 6.3]
